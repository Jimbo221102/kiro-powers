# 進捗チェック手順

このファイルはKiroが進捗確認を実行する際の具体的な手順と判定ロジックを定義します。
**目的は「チームの開発状況を正確に把握し、滞留や抜け漏れを検出すること」です。** そのため、情報は必ずリモート（origin）の状態を基準に判断してください。ローカルの状態は参考程度に留めます。

---

## 情報収集の手順

情報収集は以下の順番で実施します。後のステップほど詳細な情報が必要になるため、まず全体像を把握してから個別に掘り下げます。

### 1. リモートのfeatureブランチ一覧を取得

最新のリモート状態を取得してから一覧を確認します（古いキャッシュで判断しないため）。

```bash
git fetch --all --prune
git branch -r | grep "feature/"
```

### 2. developにマージ済みのブランチを特定

マージ済みブランチを先に特定しておくことで、後の判定が効率的になります。

```bash
git branch -r --merged origin/develop | grep "feature/"
```

### 3. Open状態のMR一覧を取得

全ブランチのMR状況を一括取得します（ブランチごとに個別取得すると時間がかかるため）。

```bash
glab mr list --state opened
```

### 4. 各ブランチのコミット数確認

developより先行しているコミット数を確認します。コミットがない場合はブランチ作成直後か、developと同期済みです。

```bash
git log origin/develop..origin/feature/xxx --oneline
```

### 5. 滞留検出のための最終コミット日時確認

Step 3（リモートpush済み）のままMRが作られていないブランチは滞留の可能性があります。

```bash
git log -1 --format="%ci %s" origin/feature/xxx
```

---

## ステップ判定ロジック

各ブランチについて **Step 5から逆順に判定し、最初にYESになったステップを現在のステップとして報告します。** これにより、最も進んでいる工程が正確に特定できます。

```
【Step 5: マージ済み】
  条件: git branch -r --merged origin/develop にブランチが含まれる
  → YES → Step 5 完了。以降の判定は不要。

【Step 4: MR作成済み】
  条件: glab mr list の結果にこのブランチのMRが存在する（Open or Merged）
  → YES → Step 4 完了。MR番号とタイトルを記録。

【Step 3: リモートpush済み】
  条件: git branch -r にブランチが存在する
  → YES → Step 3 完了。
  → この時点でMRがない場合は滞留候補としてマーク（⚠️）

【Step 2: 実装中】
  条件: git log origin/develop..origin/<branch> にコミットが1件以上ある
  → YES → Step 2 完了。コミット数を記録。

【Step 1: ブランチ作成】
  条件: ブランチが存在する時点で完了
  → 常にYES（ブランチが存在する前提）
```

## 進捗レポートの出力形式

収集した情報をもとに、以下の形式でレポートを生成してください。**サマリーテーブルを先に出し、詳細は後に続けます。** これにより、チームが全体像を素早く把握できます。

```
## featureブランチ 進捗サマリー（YYYY-MM-DD時点）

| ブランチ名 | 現在のステップ | 工程名 | 備考 |
|-----------|-------------|--------|------|
| feature/aaa | Step 4 | MR作成済み | MR #12 Open中 |
| feature/bbb | Step 3 | リモートpush済み | MRなし ⚠️ |
| feature/ccc | Step 5 | マージ済み | develop取込済み |
| feature/ddd | Step 2 | 実装中 | コミット3件 |

### ⚠️ 要確認ブランチ
- **feature/bbb**: push済みだがMRが作成されていません（最終コミット: YYYY-MM-DD）

---

### 詳細

**feature/aaa**
- リモート存在: ✅
- developより先行コミット: 5件
- MR: ✅ #12「〇〇機能追加」（Open）
- developマージ済み: ❌
→ **現在のステップ: Step 4（MR作成済み）**

**feature/bbb**
- リモート存在: ✅
- developより先行コミット: 2件
- MR: ❌ なし
- developマージ済み: ❌
- 最終コミット: 2026-05-10（7日前）
→ **現在のステップ: Step 3（リモートpush済み）** ⚠️ MR未作成
```

---

## 滞留ブランチの判定基準

以下の条件に該当するブランチは「⚠️ 要確認」としてレポートに追記します。
チームが見落としを防ぐためのアラートです。

| 条件 | 判定 |
|------|------|
| Step 3（push済み）のままMRがない | ⚠️ MR未作成 |
| Step 4（MR作成済み）のまま長期間経過 | ⚠️ レビュー待ち長期化（最終コミット日で判断） |
| コミットが0件のままリモートに存在 | ⚠️ 実装未着手の可能性 |
